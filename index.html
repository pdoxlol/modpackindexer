curseforge links are annoying on purpose, apparently

  
modrinth links can load automatically because modrinth lets a static browser page use its public api. curseforge is different: its official api needs a private api key, and putting that key inside this html file would expose it to literally everyone.

  
for curseforge packs, download the modpack .zip from curseforge first, then drop it into the upload box above. the site can read the pack’s manifest.json locally in your browser.

  
fully automatic curseforge link importing would need a tiny backend/serverless proxy, like cloudflare workers, netlify functions, or vercel functions.
